Übung: Cube View mit eingeschränkten Kennzahlen und Zählern

Erweiterung eines Calculation View vom Typ CUBE mit eingeschränkten Kennzahlen.

Aufgabenbeschreibung:

Erweitern Sie den Cube View CV_CUBE_CALC aus vorheriger Übung um eingeschränkte Kennzahlen und Zähler.

Folgende Kennzahlen sollen im View enthalten sein:

  • IST-Aufwand aller Aufgaben, die in Bearbeitung sind

    • Basiskennzahl: EFFORT
    • Einschränkung auf STATUS = 3

  • Geplanter Aufwand der Aufgaben in den nächsten zwei Wochen

    • Basiskennzahl = PLANNED_EFFORT
    • Einschränkung auf DUE_DATE: DUE_DATE >= 01.01.2018 (festes Datum) und DUE_DATE < = 15.01.2018

  • Zähler für alle Aufgaben (TASKID)

  • Zähler für alle Bearbeiter von Aufgaben (ASSIGNEE)

Führen Sie Auswertungen nach verschiedenen Dimensionen durch:

  • Aufwände nach Mitarbeitern
  • Aufwände nach Projekt

Vorgehen

Öffnen Sie den Cube View CV_CUBE_CALC aus vorheriger Übung.

Klicken Sie in den Aggregationsknoten und wählen im Reiter Output “Restricted Columns”:

Wählen Sie “New”. Füllen Sie das Dialogfenster wie folgt:

IST-Aufwand aller Aufgaben, die in Bearbeitung sind:

  • Name: EFFORT_IN_PROCESS
  • Column: EFFORT
  • Restrictions:
  • STATUS = 3

Aktivieren Sie den View und führen die Datenvorschau aus.

Betrachten Sie die Datenvorschau auf Aufgabenebene (Spalte TASKID).

Die Kennzahl zeigt nur Werte für Aufgaben mit STATUS = 3 an.

Kennzahl nach Zeitraum einschränken

Als nächstes definieren Sie eine Kennzahl, die nur Datensätze aus einem bestimmten Zeitraum betrachtet.

Gehen Sie analog zu oben vor.

Legen Sie eine eingeschränkte Kennzahl an:

  • Name: PLANNED_EFFORT_BY_DUE_DATE
  • Spalte DUE_DATE
  • Einschränkung auf DUE_DATE: DUE_DATE >= 01.01.2018 (sic!) und DUE_DATE ⇐ 15.01.2018

Sie können die Einschränkung als Selektion mitgeben. Analog zu den Selektionsoptionen im BW (Klick auf die drei Punkte in Spalte Value):

Zusatz: Das zweite Datum für das Intervall können Sie auch mit einem SQL-Ausdruck berechnen:
